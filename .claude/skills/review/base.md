# Review Base (공통 프레임워크)

모든 review 프로필이 공유하는 파일 수집 로직과 출력 포맷.

## 파일 수집

우선순위대로:

1. **인자로 파일 경로가 주어지면** 그 파일만.
2. **main 대비 브랜치 전체 변경 + 미커밋 변경**:
   ```bash
   git diff --name-only origin/main...HEAD
   git diff --name-only HEAD
   git diff --name-only --cached
   ```
3. **main과 차이가 없으면** staged + unstaged 변경 파일만.

수집된 파일 중 각 프로필의 **파일 필터 패턴**에 매칭되는 것만 리뷰한다. 매칭이 없으면 그
프로필은 건너뛴다. `node_modules`, 빌드 산출물(`out/`, `.next/`, `dist/`, `storybook-static/`)은
제외한다.

## 심각도

| 심각도 | 기준 |
|--------|------|
| **Critical** | 빌드/배포 실패, 크롤링 차단, 데이터 손실, 잘못된 라우팅(404), 규칙 정면 위반 |
| **Warning** | 품질/일관성 저하, 접근성/성능 문제, 중복, 계층 위반 |
| **Suggestion** | 추가 최적화, 구조화 확장, 사소한 개선 |

## 출력 포맷

프로필별로 아래 구조를 따른다.

```markdown
## {프로필명} Review

### {파일 경로}:{줄}
- **[심각도] 한 줄 요약**
  - 근거: 무엇이 왜 문제인가 (실패 시나리오 또는 규칙)
  - 수정안: 구체적으로 어떻게 고칠지 (가능하면 코드/값)
```

문제가 없으면 "위반 없음"으로 명시한다. 추측이면 추측이라고 표시한다.

### SEO / GEO 추가 출력

SEO와 GEO 프로필은 위 형식에 더해 **카테고리별 점수 요약**과 **우선순위 액션 플랜**을
낸다.

```markdown
### 요약
| 카테고리 | 점수 | 상태 |
|----------|------|------|
| ... | X/Y | OK / 주의 / 위험 |

### 액션 플랜
#### P0 (지금)
1. ...
#### P1 (곧)
1. ...
#### P2 (나중)
1. ...
```

## 주의

- 코드 분석 기반이며 실제 크롤러/런타임 테스트는 별도. Core Web Vitals와 AI 인용 여부는
  추정만 가능하다고 밝힌다.
- 이 저장소의 표준 규칙(루트 `CLAUDE.md`)을 항상 기준으로 삼는다. 특히 긴 대시 금지, DS 토큰,
  이중언어 로케일 링크(`localePath`)와 hreflang, GitHub 비노출.
